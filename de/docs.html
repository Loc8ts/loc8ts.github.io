Erste Schritte

Alles, was du brauchst, um Loc8ts selbst zu hosten und die Ersteinrichtung abzuschließen.

In aktiver Entwicklung

Loc8ts steht noch vor Version 1.0.0 und entwickelt sich schnell weiter. Jede Installation davor gilt als Wegwerf-Installation: Datenbanken aus Pre-1.0.0-Versionen werden nicht migriert - rechne also damit, zwischen Updates neu aufzusetzen.

Voraussetzungen

  • Docker mit dem Compose-Plugin (in Docker Desktop bereits enthalten, unter Linux docker-compose-plugin nachinstallieren).
  • Eine Domain, die auf deinen Server zeigt - für HTTPS erforderlich.
  • Ein Reverse-Proxy für die SSL-Terminierung (Caddy, nginx, Traefik oder Nginx Proxy Manager).

Mit Docker installieren

Loc8ts läuft als einzelner Container - ganz ohne externe Datenbank. Lege eine docker-compose.yml neben einem Datenverzeichnis an:

Stelle TZ auf deine eigene Zeitzone um (etwa America/New_York oder Asia/Tokyo). Sie bestimmt, wann geplante Backups laufen - beim Standardwert richten sie sich nach UTC.

Umgebungsvariablen

VariableStandardBeschreibung
TZUTCLokale Zeitzone, bestimmt die Zeitpläne der Backups. Standard ist UTC.
ENABLE_ADMIN_UI-Auf "1" setzen, um das PocketBase-Admin-Panel unter /_/ freizugeben. Nur für Wartungsarbeiten aktivieren.

Dann starten:

docker compose up -d

Beim ersten Start verlangt Loc8ts einen einmaligen Setup-Token, bevor du das Admin-Konto anlegen kannst. Er landet in den Container-Logs und in data/setup_token.txt:

docker compose logs -f loc8ts
# or read it from the data directory:
cat data/setup_token.txt

Ersteinrichtung

Öffne die App im Browser. Ein Assistent führt dich durch die Absicherung der Installation und das Anlegen deines Admin-Kontos.

  1. 1. Installation verifizieren

    Kopiere den Setup-Token aus data/setup_token.txt oder den Container-Logs und füge ihn ein. Nach Abschluss der Einrichtung wird der Token ungültig und die Datei automatisch gelöscht.

  2. 2. Anwendung konfigurieren

    Ein einziger Schritt fragt dein Admin-Konto sowie optionale SMTP-, Signup- und Backup-Einstellungen ab. Alles Optionale kannst du später im Admin-Bereich anpassen.

  3. 3. Admin-Konto anlegen

    Gib Name, E-Mail und ein Passwort mit mindestens 8 Zeichen ein - das wird dein erstes Administratorkonto.

  4. 4. SMTP einrichten (optional)

    Aktiviere SMTP, damit Loc8ts Einladungen, E-Mail-Verifizierungen, Passwort-Resets und Benachrichtigungen verschicken kann. Trage Server, Zugangsdaten, Absender und die öffentliche App-URL ein.

  5. 5. Test-E-Mail senden

    Schick dir eine Test-E-Mail, um die Verbindung zu prüfen. Klappt der Test, werden darunter auch die öffentlichen Signups freigeschaltet.

  6. 6. Öffentliche Signups (optional)

    Erlaube Nutzern, sich selbst zu registrieren. Dafür ist SMTP nötig: Jede Registrierung muss zuerst über einen Link per E-Mail bestätigt werden, und neue Konten starten immer mit der schreibgeschützten Viewer-Rolle. Lege fest, wie ein Konto nach der Bestätigung aktiviert wird:

    Selbstverifizierung
    Mit dem Klick auf den Bestätigungslink ist das Konto sofort aktiv - mit Viewer-Zugriff. Admins werden über den neuen Nutzer informiert und können die Rolle später bei Bedarf anheben.
    Admin-Freigabe
    Auch nach der E-Mail-Bestätigung bleibt das Konto gesperrt, bis ein Admin es unter Admin -> Users freigibt. Admins bekommen dazu eine Zugriffsanfrage per E-Mail - anmelden kann sich die Person erst nach der Freigabe.

  7. 7. Geplante Backups (optional)

    Plane automatische Backups nach Häufigkeit und Uhrzeit - auf Wunsch werden nur die letzten N behalten. Backups landen in data/backups.

  8. 8. Fertigstellen

    Leg das Admin-Konto an, um die Einrichtung abzuschließen. Version und Speicherbelegung findest du jederzeit im System-Panel des Admin-Bereichs.

Aktualisieren

Zieh das neue Image und starte den Container neu - deine Daten bleiben im eingebundenen Datenverzeichnis erhalten. Wirf vor dem Upgrade einen Blick in die Release-Notes, ob es Breaking Changes gibt.

Zum Changelog 
docker compose pull
docker compose up -d

Backups

Der gesamte Zustand - Datenbank, Uploads und Backups - liegt unter /app/data. Sichere dieses Verzeichnis direkt oder nutze die integrierten geplanten Backups aus dem Admin-Bereich.

Es gibt zwei Arten von Backups, in der App am auto- bzw. manual-Pill zu erkennen. Geplante Backups entstehen automatisch, manuelle löst du selbst aus. Auf der Festplatte unterscheiden sie sich am Dateipräfix:

Die Aufbewahrungsgrenze (letzte N behalten) räumt nur auto-Backups auf. Manuelle Backups sind von der automatischen Löschung ausgenommen und bleiben, bis du sie selbst entfernst.
