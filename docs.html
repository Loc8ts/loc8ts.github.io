Getting started

Everything you need to self-host Loc8ts and get through first-run setup.

Under active development

Loc8ts is still pre-1.0.0 and moving fast. Treat any installation before v1.0.0 as disposable: pre-v1.0.0 databases aren't migrated, so expect to wipe and start fresh between updates.

Requirements

  • Docker with the Compose plugin (bundled with Docker Desktop, or install docker-compose-plugin on Linux).
  • A domain name pointing at your server - required for HTTPS.
  • A reverse proxy for SSL termination (Caddy, nginx, Traefik or Nginx Proxy Manager).

Install with Docker

Loc8ts runs as a single container - no external database needed. Create a docker-compose.yml next to a data directory:

Set TZ to your own timezone (for example America/New_York or Asia/Tokyo). It decides when scheduled backups run - left at the default, they follow UTC time.

Environment variables

VariableDefaultDescription
TZUTCLocal timezone, used to schedule backups. Defaults to UTC.
ENABLE_ADMIN_UI-Set to "1" to expose the PocketBase admin panel at /_/. Leave it off unless you're doing maintenance.

Then start it:

docker compose up -d

The first time it starts, Loc8ts asks for a one-time setup token before you can create the admin account. You'll find it in the container logs and in data/setup_token.txt:

docker compose logs -f loc8ts
# or read it from the data directory:
cat data/setup_token.txt

First-run setup

Open the app in your browser. A guided wizard walks you through securing the install and creating your admin account.

  1. 1. Verify your installation

    Grab the setup token from data/setup_token.txt or the container logs, paste it in and continue. Once setup completes, the token stops working and setup_token.txt is deleted automatically.

  2. 2. Configure your application

    One screen covers it all: your admin account plus optional SMTP, signup and backup settings. Anything optional can be changed later in the Admin section.

  3. 3. Create the admin account

    Enter a name, an email and a password of at least 8 characters - this becomes your first administrator.

  4. 4. Configure SMTP (optional)

    Enable SMTP so Loc8ts can send invites, email verification, password resets and notifications. Fill in your server, credentials, sender and the public App URL.

  5. 5. Send a test email

    Send yourself a test email to make sure the connection works. A successful test also unlocks public signups below.

  6. 6. Public signups (optional)

    Let people register their own accounts. This requires SMTP: every signup has to confirm an email verification link first, and new accounts always start out with the read-only Viewer role. Choose how an account is activated once it's verified:

    Self-verification
    Clicking the verification link activates the account right away, with Viewer access. Admins are notified about the new user and can raise their role later if needed.
    Admin approval
    Even after the email is verified, the account stays locked until an admin approves it under Admin -> Users. Admins get an access-request email, and the person can only sign in once approved.

  7. 7. Scheduled backups (optional)

    Schedule automatic backups by frequency and time of day, and optionally keep just the last N. Backups end up in data/backups.

  8. 8. Finish

    Create the admin account to wrap up setup. You can check version and storage details any time from the System panel in Admin.

Put it behind HTTPS

Loc8ts needs HTTPS for camera and clipboard features. Terminate SSL with a reverse proxy - for example Caddy:

nginx, Traefik or Nginx Proxy Manager work just as well. Whatever you pick: never expose port 8090 directly to the internet.

Updating

Pull the new image and restart - your data stays safe in the mounted data directory. Give the release notes a quick look for breaking changes before you upgrade.

View the changelog 
docker compose pull
docker compose up -d

Backups

All state - database, uploads and backups - lives under /app/data. Back that directory up directly, or use the built-in scheduled backups from the Admin section.

Backups come in two flavors, marked with an auto or manual pill in the app. Scheduled backups are created automatically - manual ones are those you trigger yourself. On disk, you can tell them apart by their prefix:

The retention limit (keep last N) only prunes auto backups - manual backups are never deleted automatically and stick around until you remove them yourself.
